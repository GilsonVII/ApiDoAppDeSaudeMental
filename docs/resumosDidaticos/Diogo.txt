[Pool]: Conjunto de recursos pré-alocados, prontos para serem usados. O objetivo de usar um pool é melhorar a performance e a eficiência da aplicação, evitando o tempo e o custo de criar e destruir recursos repetidamente.

Pool de Conexões de Banco de Dados:
O que é: Um conjunto de conexões de banco de dados que já foram abertas e autenticadas.

Por que usar: Abrir e fechar uma conexão de banco de dados para cada solicitação é lento e consome muitos recursos do servidor de banco de dados. Um pool mantém as conexões ativas, prontas para serem "emprestadas" à API quando necessário, e as retorna ao pool após o uso. Isso reduz a latência e o consumo de recursos.

[Async]
Toda função declarada com async tem duas características principais:

Sempre Retorna uma 'Promise': Mesmo que você retorne um valor simples (como return 5;), a função async irá automaticamente envolver esse valor em uma Promise resolvida. Se a função lançar um erro, a Promise será rejeitada.
Permite o Uso de await: O async é o que permite que você use a palavra-chave await dentro do corpo da função.

"Await" faz a função esperar até um valor ser retornado, usado frequentemente em operações demoradas (operação BD).
"Promise" é um objeto que representa a eventual conclusão (ou falha) de uma operação async e seu valor resultante.

{ZOD}
Em resumo, ela garante a segurança de tipos em tempo de execução, garantindo que os dados que seu código recebe são exatamente o que você espera.

Quando você envia dados através de parâmetros URL, é sempre recebido pelo servidor como String.
Na lógica do banco de dados, os dados como IDs devem ser tratados como Numbers. O Zod garante que, após a validação, a aplicação trabalhará com o tipo de dado correto (number)

[ctx] => É um objeto especial que o Zod fornece dentro da função .transform() ou .refine(), permitindo que você interaja com o processo de validação. Ele é crucial para adicionar mensagens de erro personalizadas quando uma validação falha em uma lógica de transformação.

[regex] => garante que a string recebida esteja no formato esperado.
data_hora: z.string().regex(/^\d{2}:\d{2}:\d{2}$/, "Formato de data_hora deve ser HH:MM:SS");
